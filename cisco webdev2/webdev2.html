CSS Basics

Introduction

CSS Rule is a set of instructions that defines how a specific HTML element should be styled and presented.

It is composed of "selector" and its associated declaration block.

A declaration block is a set of properties and their corresponding values enclosed in {curly braces}

CSS Comments

Just use /* these characters */ for commenting.

Every programmer, even students, know why we use comments.

"Comments are essential for documenting your code, providing context, and explaining design decisions."

Element Selectors

The most basic form of CSS selector is the element selector, also known as the type selector. It allows you to target a specific HTML element to apply styles to, based on that element tag name. The styles you apply will affect all instances of that specific HTML element on your web page.

Class Selectors

Class selectors allow you to target HTML elements with a specific class attribute. They are denoted by a period (.) followed by the class name.

Unlike element selectors, class selectors provide a higher level of specificity

ID Selectors

ID selectors target a single HTML element with a unique ID attribute. They are denoted by a hash symbol (#) followed by the ID name.

Class selectors are versatile in applying consistent styles to multiple elements, while ID selectors provide specificity by focusing on a solitary and distinctive element.

Attribute Selectors

Attribute selectors allow you to target elements based on their attributes. They're like filters for elements with specific attributes.

There are different types of Attribute Selector:

  • Exact Match Selector

    • This selector targets elements with an attribute that exactly matches a value.

    [attribute="value"] { /* styles */ }

  • Substring Match Selector

    • This selector targets elements with an attribute containing a specified value.

    [attribute*="value"] { /* styles */ }

  • Prefix Match Selector

    • This selector targets elements with an attribute starting with a value.

    [attribute^="value"] { /* styles */ }

  • Suffix Match Selector

    • The selector targets elements with an attribute ending with a value.

    [attribute$="value"] { /* styles */ }

    CSS Cascade

    CSS cascade determines which rule takes precedence.

    In general, it follows a specific order: inline styles have the highest precedence, followed by IDs, classes, and element selectors.

    !important

    The !important declaration is used to give a style higher precedence, overriding other styles (even those with higher specificity!). It's like a trump card that can disrupt the natural order of the cascade, where styles are typically applied in a specific sequence based on factors like specificity and order.
